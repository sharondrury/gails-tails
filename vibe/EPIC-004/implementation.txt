Feat-002 | Accordion layout — mobile only
  Two separate markup sections in PriceList.jsx:

  .price-list__desktop — wraps the existing tab layout
    Shown above 850px (default display: block)
    Hidden below 850px via SCSS media query (display: none)

  .price-list__mobile — wraps the new accordion layout
    Hidden above 850px (default display: none)
    Shown below 850px via SCSS media query (display: block)

  Data consolidation: infoData object and buttons array merged into a single
  items array. Each item has: key, heading, subheading, content.
  Subheading is a short descriptor shown in the collapsed accordion header.

  Desktop tab layout preserved exactly — only wrapped in .price-list__desktop div.
  key prop added to .price-list__fade div (keyed on activeButton) to re-trigger
  the fade animation on tab change.

  Engine components considered: None applicable.
  Files changed:
    - MODIFIED: src/components/PriceList.jsx (data consolidated, desktop wrapped,
      mobile accordion markup added)
    - MODIFIED: src/components/PriceList.scss (desktop/mobile show/hide rules,
      full accordion SCSS at 850px breakpoint)

  What was added (learning):
    - Two sibling layout sections (desktop + mobile), shown/hidden by CSS, is
      a clean pattern for breakpoint-driven layout switches. No JS media query
      detection needed — CSS handles the visibility, React handles the state.

Feat-004 | One open at a time
  State added to PriceList component:
    const [activeIndex, setActiveIndex] = useState(null)

    const handleAccordionToggle = (i) => {
      setActiveIndex((prev) => (prev === i ? null : i))
    }

  Toggle function: if tapping already-open item → closes it (null).
  If tapping a different item → closes previous, opens new one.
  Single state value ensures exclusivity automatically.

  Engine components considered: None applicable.
  Files changed:
    - MODIFIED: src/components/PriceList.jsx (activeIndex state, handleAccordionToggle)

Feat-005 | Smooth expand/collapse transition
  CSS max-height + opacity transition in PriceList.scss:

  .price-list__item-body (collapsed default):
    max-height: 0; overflow: hidden; opacity: 0;
    transition: max-height 0.35s ease, opacity 0.25s ease;

  .price-list__item--open .price-list__item-body (expanded):
    max-height: 600px; opacity: 1;

  The --open modifier class is applied in JSX:
    className={`price-list__item${activeIndex === i ? " price-list__item--open" : ""}`}

  Also: inline <style jsx> animation from the original component removed and
  moved to PriceList.scss as @keyframes priceListFadeIn on .price-list__fade.
  Renamed from fadeIn to priceListFadeIn to avoid collision with global keyframes.

  Engine components considered: None applicable.
  Files changed:
    - MODIFIED: src/components/PriceList.jsx (--open class, removed inline style tag)
    - MODIFIED: src/components/PriceList.scss (max-height transition, keyframes moved)
