Feat-005 | Fix carousel index bug
  Root cause: numScroll={3} — each arrow click advanced 3 array positions.
  With 4 items this jumped from index 0 to index 3 (id 1 to id 4), skipping ids 2 and 3.

  Fix: numScroll={1} on the Carousel component.

  Additional cleanup:
    - responsiveOptions array removed — all breakpoints want numVisible=1, numScroll=1
      which matches the top-level props, making responsive overrides redundant
    - Data array variable renamed from ProductService to testimonials
    - State variable renamed from products to items for clarity

  Engine components considered: None applicable.
  Files changed:
    - MODIFIED: src/components/testimonials/testimonials.jsx (numScroll fix, data rename)

  What was added (learning):
    - PrimeReact Carousel numScroll controls how many items move per click.
      numScroll={3} with 4 items caused the jump-to-last bug. Always set
      numScroll={1} for a standard one-at-a-time carousel.
