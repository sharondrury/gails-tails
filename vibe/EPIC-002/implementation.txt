Feat-003 | homePage.jsx composition
  Import order inside homePage.jsx:

    import Nav          from '../../../components/nav/nav'
    import Hero         from '../../../components/hero/hero'
    import AboutPage    from '../../../components/About'
    import PriceList    from '../../../components/PriceList'
    import Testimonials from '../../../components/testimonials'
    import ContactUs    from '../../../components/contactUs/contactUs'
    import './homePage.scss'
    import '../../../App.css'

  Component shape (following rules.txt file template):

    const HomePage = () => {
      return (
        <>
          <Nav />
          <main className="bg-[#D9D9D9] pt-[88px]">
            <div className="w-full max-w-[500px] min-[500px]:max-w-[1000px] mx-auto relative">
              <div id="home" className="h-auto p-4 scroll-mt-[88px]">
                <Hero />
              </div>
              <div id="about" className="h-auto bg-white p-4 text-gray-700 scroll-mt-[88px]">
                <AboutPage />
              </div>
              <div id="price-list" className="h-auto bg-[#F38299] p-4 scroll-mt-[88px]">
                <PriceList />
              </div>
              <div id="testimonials" className="h-auto p-4 scroll-mt-[88px]">
                <Testimonials />
              </div>
              <div id="contact-us" className="h-auto bg-white scroll-mt-[88px]">
                <ContactUs />
              </div>
            </div>
          </main>
        </>
      )
    }

    export default HomePage

  Note: layout classes (bg-[#D9D9D9], pt-[88px], scroll-mt etc.) are lifted
  directly from the existing App.jsx — no visual change, just moved responsibility.

  Scroll-target ids confirmed present:
    id="home", id="about", id="price-list", id="testimonials", id="contact-us"

  Engine components considered: None applicable — composition only.
  Files changed:
    - MODIFIED: src/website/pages/homePage/homePage.jsx

  What was added (learning):
    - A page composition file (homePage.jsx) is responsible only for assembling
      sections in order with their layout wrappers. It does not own any logic —
      each section component handles its own state and behaviour.
    - This separation means adding or reordering sections only requires a change
      in one file (homePage.jsx), not App.jsx.

Feat-004 | Wire into App.jsx
  App.jsx after this feat:

    import HomePage from './website/pages/homePage/homePage'
    import './App.css'

    function App() {
      return <HomePage />
    }

    export default App

  What is removed from App.jsx:
    - import Nav
    - import Homepage (now Hero, in homePage.jsx)
    - import AboutPage
    - import PriceList
    - import Testimonials
    - import ContactUs
    - All JSX section markup (moves into homePage.jsx)

  App.css import stays in App.jsx — it is the global CSS entry point.
  If App.css is also needed inside homePage.jsx, import it there and remove
  from App.jsx to avoid double-loading.

  Engine components considered: None applicable.
  Files changed:
    - MODIFIED: src/App.jsx (stripped to shell, now renders only HomePage)

  Tradeoffs / follow-up:
    - The src/website/ folder structure is introduced here for the first time.
      Existing components (About, PriceList, etc.) remain in src/components/ for
      now — migrating them into src/website/ is a future EPIC.
    - The Tailwind utility classes on section wrappers (bg-[...], scroll-mt-[...])
      are hardcoded in homePage.jsx. These should move to homePage.scss using
      CSS variables once the project is further along.

  What was added (learning):
    - App.jsx as a thin shell (render one component) is the standard pattern for
      React apps that grow beyond a single page. It makes routing easy to add
      later — a router wraps <HomePage /> without touching any section components.

General notes (actual implementation)
  - hero.scss was created this EPIC — did not exist before Feat-005
  - All hero styles live in hero.scss; App.css retains only :root font declaration
  - SVG imports use Vite static import syntax — resolved at build time
  - aria-hidden="true" on all three decorative paw img tags
  - z-index layering: paws z-index 0 (SCSS), dog photo z-[1] (Tailwind), hero content z-index 1 (SCSS),
    bio card z-[1] (Tailwind — needed to sit above right paw)
  - Dog photo src kept as relative path ./src/assets/images/ per user preference
  - overflow: hidden deliberately omitted from .hero and .hero__bio — dog photo and paws bleed intentionally
